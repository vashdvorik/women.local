# API и план реализации

## 1. Принципы API

Основной web-интерфейс может работать через Inertia.js, но бизнес-функции оформляются как независимые application actions. REST API используется для Telegram, интеграций и будущих клиентов.

- префикс: `/api/v1`;
- JSON в UTF-8;
- серверная валидация с кодами ошибок и локализованными сообщениями;
- пагинация через cursor для больших списков;
- идемпотентность для webhook и критичных POST-запросов;
- OpenAPI-описание для интеграционных endpoint;
- даты в ISO 8601 и UTC, отображение в локальной временной зоне;
- фильтры только из разрешённого списка.

## 2. Основные endpoint

| Область | Endpoint |
|---|---|
| Auth | `POST /auth/register`, `POST /auth/login`, `POST /auth/logout`, `POST /auth/password/forgot`; Telegram auth/confirmation добавляется после утверждения сценария |
| Profile | `GET/PATCH /me`, `GET/PATCH /me/business`, `PATCH /me/privacy` |
| Directory | `GET /directory`, `GET /directory/{profile}` |
| Learning | `GET /courses`, `GET /courses/{course}`, `POST /courses/{course}/enroll`, `PUT /lessons/{lesson}/progress` |
| Events | `GET /events`, `POST /events/{event}/registrations`, `DELETE /events/{event}/registrations/me` |
| Opportunities | `GET /opportunities`, `POST /opportunities/{opportunity}/save` |
| Marketplace | `GET/POST /posts`, `GET/PATCH /posts/{post}`, `POST /posts/{post}/responses` |
| Mentoring | `GET /mentors`, `POST /mentoring/applications`, `GET /mentorships/me` |
| Recommendations | `GET /recommendations`, `POST /recommendations/{id}/feedback` |
| Notifications | `GET /notifications`, `PATCH /notification-preferences` |
| Telegram | `POST /telegram/link-token`, `POST /webhooks/telegram` |
| Privacy | `POST /me/data-export`, `POST /me/deletion-request` |

Административные endpoint находятся в отдельной группе, требуют специальных permissions и покрываются аудитом.

## 3. События домена

- `UserRegistered`;
- `EmailVerified`;
- `ProfileSubmittedForReview`;
- `ProfileApproved`;
- `CourseCompleted`;
- `CertificateIssued`;
- `EventRegistrationCreated`;
- `PostPublished`;
- `MentorshipStarted`;
- `MentoringSessionCompleted`;
- `OpportunityPublished`;
- `TelegramLinked`;

Обработчики событий запускают индексацию, уведомления, пересчёт показателей и рекомендации. Ошибка внешнего канала не должна отменять основную бизнес-транзакцию.

## 4. Этапы реализации

Этапы соответствуют исходному плану. Сроки определяются после утверждения технического задания, прототипов, состава команды и инвентаризации контента.

### Этап 1. Концепция и техническое задание

- подтвердить владельца продукта и матрицу ответственности;
- согласовать термины «два берега», регионы и правовые роли;
- утвердить обязательные поля, правила видимости и согласия;
- определить критерии модерации и публикации;
- подготовить карту контента и переводы;
- зафиксировать состав первой версии, показатели и критерии приёмки.

Результат: утверждённая концепция платформы и ToR, backlog, карта ролей и основные пользовательские сценарии.

### Этап 2. Дизайн и архитектура

- визуальная концепция и дизайн-система;
- основные публичные страницы;
- личный кабинет и профиль;
- каталог, обучение и мобильные сценарии;
- архитектура приложения, данных, интеграций и инфраструктуры.

Результат: утверждённый дизайн-пакет, прототипы и архитектурная спецификация.

### Этап 3. Техническая разработка

- инфраструктура, CI/CD, мониторинг и резервное копирование;
- трёхъязычный публичный сайт и CMS;
- регистрация, личный кабинет, роли, согласия и приватность;
- профиль и бизнес-карточка;
- приватность и модерация;
- каталог, фильтры, поиск и избранное;
- запросы, предложения и отклики;
- курсы, уроки, прогресс и сертификаты;
- события и регистрации;
- возможности и сроки;
- Telegram/email-уведомления;
- наставники, заявки и ручное создание пар;
- аналитика и административная панель;
- базовый AI-поддерживаемый поиск и рекомендации с fallback на правила.

Результат: полный функциональный beta-релиз.

### Этап 4. Контент и тестирование

- загрузка учебных материалов, событий и возможностей;
- пилот с представительницами целевых групп;
- accessibility, security и performance testing;
- исправления по обратной связи;

Результат: протестированная и улучшенная платформа, готовая к запуску.

### Этап 5. Запуск и продвижение

- production launch и мониторинг;
- обучение команды проекта и передача регламентов;
- коммуникационная кампания;
- мобилизация партнёров и регистрация участниц.

Результат: работающая платформа и растущее сообщество.

## 5. Команда

Минимальный состав:

- business analyst / project manager;
- UX/UI designer;
- backend/full-stack developer;
- frontend developer при параллельной разработке;
- QA engineer;
- DevOps part-time;
- content manager и переводчики RU/RO/EN;
- специалист по защите данных и юридическим документам;
- координатор сообщества и наставничества.

## 6. Стратегия тестирования

| Уровень | Что проверяется |
|---|---|
| Unit | Правила доступа, статусы, score рекомендаций, прогресс и сертификаты |
| Integration | База данных, поиск, очереди, Telegram, email, файловое хранилище |
| Feature | Регистрация, профиль, публикации, обучение, события и наставничество |
| E2E | Главные сценарии участницы, модератора и администратора |
| Security | Права доступа, IDOR, rate limiting, загрузка файлов, сессии, webhook |
| Accessibility | Клавиатура, focus, labels, contrast, screen reader smoke tests |
| Performance | Каталог, поиск, dashboard, массовые уведомления и экспорт |

Критичные сценарии тестируются на всех трёх языках и на типичном мобильном экране.

## 7. Критерии приёмки первой версии

- пользователь регистрируется, подтверждает email и управляет согласиями;
- профиль проходит модерацию и корректно соблюдает видимость полей;
- каталог фильтрует по региону, сектору, стадии, интересам и потребностям;
- курсы сохраняют прогресс и выдают сертификат только после выполнения условий;
- события соблюдают лимит мест и предотвращают двойную регистрацию;
- запросы и предложения публикуются по правилам модерации;
- заявки на наставничество проходят весь согласованный workflow;
- email и Telegram учитывают пользовательские настройки;
- администратор управляет контентом без участия разработчика;
- отчёты показывают регистрации, заполненность профилей, активность, обучение, события, публикации и наставничество;
- базовый AI-поддерживаемый поиск и рекомендации могут быть отключены без нарушения основных пользовательских сценариев;
- интерфейс работает на RU/RO/EN и основных мобильных разрешениях;
- резервная копия успешно восстанавливается на тестовой среде;
- действия администраторов доступны в журнале аудита.

## 9. Риски и меры

| Риск | Мера |
|---|---|
| Неполные профили | Пошаговое заполнение, индикатор прогресса, напоминания |
| Мало активности | Посев контента, координаторы, амбассадоры, регулярные дайджесты |
| Утечка контактов | Скрытие по умолчанию, политики доступа, отклики внутри платформы |
| Слабое качество рекомендаций | Сначала прозрачные правила, сбор обратной связи, затем AI |
| Перегрузка модераторов | Очереди, приоритеты, шаблоны решений, доверенные авторы |
| Некачественные переводы | Workflow статусов перевода и ответственные редакторы |
| Зависимость от поставщика AI | Абстракция провайдера и возможность полного отключения |
| Низкая цифровая грамотность | Простые формы, mobile-first, подсказки и поддержка координатора |
| Устаревшие возможности | Обязательный срок действия и автоматическая архивация |
| Недостоверные показатели | Единые определения метрик и журнал происхождения данных |

## 10. Решения, которые нужно утвердить до разработки

1. Юридический владелец и оператор персональных данных.
2. Страны хранения базы, файлов, резервных копий и AI-обработки.
3. Какие поля профиля могут быть публичными в интернете.
4. Нужна ли ручная проверка каждой участницы и подтверждение бизнеса.
5. Кто публикует контент и отвечает за каждый язык.
6. Правила выдачи сертификатов и проверки посещаемости.
7. Формат наставничества, доступность экспертов и ответственность координатора.
8. Допустимые каналы и частота уведомлений.
9. Состав донорских отчётов и правила обезличивания.
10. Бюджет инфраструктуры, поддержки и обновления контента после запуска.
